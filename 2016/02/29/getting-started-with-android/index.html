Getting Started With Android Development

Posted by Prashant Srivastav on February 29, 2016

In January, i started exploring android development with some basic of java. I spent a lots of time on the web development and designing side of things, it occurred to me that Android is an extremely powerful platform (and open source). Before we start with the development you need to first verify that your computer meets the basic system requirements. If you’re on a mac like me make sure you have Mac OS 10.*.* or later (x86 only).

Is JDK installed?

Mac’s usually comes with the JDK preinstalled and ready to use.

For check if you have the JDK (Java development kit) installed open up terminal and type in the command "java -version". Here is output looks like.

if you don't have java installed on your system. you can download and install it.

Which IDE is better for android development

An IDE (Integrated Development Environment) is solution to develop and design app repeatedly and quickly. The use of IDE is your personal preference. But personally if I had to choose, Android Studio is widely known for android development. With Android Studio you have ADT tools integrated, and with eclipse you need to integrate them manually. With Android Studio, it feels like a tool designed from the outset with Android development in mind.

Android studio 2.0 Beta is latest release. check new feature and download beta version for stable version

For eclipse you need to install ADT plugin. To do this start eclipse go to Help > Install new software. Click Add on the top right corner. Enter the following URL: https://dl-ssl.google.com/android/eclipse/ Check developer tools, click Next, and walk through the wizard to install the tools.

Start android project!

For android developer there are a lots of Samples given by google developer team you can use it for learning purpose. for sample project is fully functioning Android app. You can browse the resources, source files and see the overall project structure. You can copy and paste the code you need

To import a code sample into Android Studio:
  • In the Android Studio menu, select File > New > Import Sample to open the Import Sample wizard.
  • Select a sample to import and click Next.
  • Specify the application name and project location if different from the displayed settings.
  • Click Finish.
Working with new project

To create new project in android studio.

  • In android studio menu, select File > New > New Project
  • Configure your new project:
    • Application Name: Name of application which you want to start with.
    • Company Domain: Java recommends that use your company's Internet domain name (which is unique)
    • Package: Java recommends that use your company domain written in reverse. Then use subpackages for different projects
    • Project Location: Where you want to save this project.

  • Next Dialog for adding a Activity in your application


  • Choose activity based on your requirement and name of activity

Understanding the project structure
  • Source (src): This folder contains all the source files i.e. this is where all the code is.
  • Res: This is the area where you keep your images, strings and xml layouts.
  • Assets: This is another place where you can keep files. The difference between /res and /assets is that Android does not generate IDs for the assets. What this means is that to access a file in assets the developer has to provide a complete path to that file.
